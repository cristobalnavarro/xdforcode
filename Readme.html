XDForCode

IDE estilo VS Code con integración nativa de agentes de IA para Windows. Desarrollado para Harbour / FiveWin.

1 ¿Qué es XDForCode?

XDForCode es un entorno de desarrollo integrado (IDE) para Windows con apariencia y funcionamiento similar a VS Code. Está diseñado especialmente para desarrolladores Harbour / FiveWin, aunque puede editar cualquier tipo de fichero.

Su característica principal es la integración nativa con agentes de IA: puedes conversar directamente con Claude, Gemini, OpenCode, Ollama y otros desde dentro del propio IDE, con acceso automático al código que estás editando.

3 Primeros pasos

Al iniciar XDForCode verás el editor vacío. Los pasos básicos para empezar:

  1. Abre una carpeta de trabajo desde el menú o haciendo clic en el icono de explorador en la barra lateral. La carpeta seleccionada aparece en el árbol de archivos.
  2. Abre un fichero: haz clic sobre él en el árbol. Se abre en el editor Monaco.
  3. Selecciona un agente de IA: en el panel derecho elige el agente que quieres usar (Claude, Gemini, Ollama…) y escribe tu consulta.
  4. Activa el servidor web (opcional): si quieres acceder desde el navegador, pulsa el icono del servidor en la barra lateral.

4 Editores — Scintilla y Monaco

XDForCode dispone de dos motores de edición que se usan según el tipo de fichero:

Monaco Editor

El mismo motor que utiliza VS Code internamente. Se emplea para la mayoría de ficheros de código y texto. Ofrece resaltado de sintaxis moderno, autocompletado y vista previa de ficheros web.

Tipos de fichero habituales: .prg, .ch, .prw, .js, .html, .css, .json, .md, .ini, .txt…

Scintilla

Editor nativo de Windows, más ligero. Se usa para ficheros que se abren en sus propias pestañas del IDE, con resaltado optimizado para Harbour y FiveWin.

Comportamiento al abrir un fichero

Tipo de ficheroComportamiento
Texto y código (.prg, .js, .txt…)Se abre en el editor (Monaco o Scintilla según tipo)
Ficheros HTMLPuedes elegir: editor de código o panel de vista previa
Ficheros binarios (.exe, .dll, imágenes…)Se abre con la aplicación asociada de Windows
Cada fichero abierto ocupa su propia pestaña en la zona de edición. Puedes tener varios ficheros abiertos a la vez y navegar entre ellos.

4 Agentes de IA — instalación recomendada

XDForCode funciona con agentes externos que se instalan en tu sistema. La aplicación los detecta y los invoca automáticamente. A continuación los recomendados:

Claude Code Nube

El agente de Anthropic. Muy potente para razonamiento y edición de código.

npm install -g @anthropic-ai/claude-code
claude
OpenClaude Nube

Fork de Claude Code con funcionalidades ampliadas. Idéntico en uso.

npm install -g openclaude
OpenCode Nube

Agente de código abierto compatible con múltiples proveedores (Anthropic, OpenAI, Google…).

npm install -g opencode
opencode auth
MIMO Nube

Fork de OpenCode. Se instala y usa exactamente igual.

npm install -g mimo
Gemini CLI Nube

El agente de Google. Contexto de hasta 1 millón de tokens.

npm install -g @google/gemini-cli
gemini auth
Ollama Gratuito · Local

Ejecuta modelos de IA en tu propio ordenador, sin internet y sin coste.

ollama pull llama3.2
ollama pull qwen2.5-coder
¿Sin cuenta, sin internet? Instala Ollama. Es la única opción completamente gratuita, sin API key y que funciona sin conexión.

5 Activar un agente en la aplicación

Una vez instalados los agentes, actívalos dentro de XDForCode:

  1. Haz clic en el selector de agente en la parte superior del panel de chat (sidebar derecho).
  2. Elige el agente de la lista: OpenCode, MIMO, Claude CLI, OpenClaude, Gemini CLI, Ollama…
  3. El panel de chat queda listo para recibir mensajes con el agente seleccionado.
Puedes tener varios agentes activos simultáneamente usando el sistema Kanban (ver sección 9).

7 Usar el panel de chat

El panel de chat (sidebar derecho) es el corazón de la integración con IA.

Enviar un mensaje

Escribe tu pregunta o instrucción en el cuadro inferior y pulsa Enviar o Ctrl+Enter.

Contexto automático

El agente recibe automáticamente el contenido del fichero que estás editando. No necesitas copiar y pegar el código.

Comandos de barra /

Escribe / en el cuadro de chat y aparecerá automáticamente la lista de todos los comandos disponibles.

Empieza por /help — es el primer comando que debes probar. Te mostrará una explicación completa de todos los comandos y opciones del chat.
ComandoAcción
/helpMuestra la ayuda completa del chat y todos los comandos disponibles
/modeCambiar el agente o modo activo
/ollamaListar y cambiar modelos de Ollama
/clearLimpiar el historial del chat
/skillInsertar una skill en el mensaje

Cancelar

Mientras el agente responde aparece el botón Cancelar. Púlsalo para interrumpir la respuesta en cualquier momento.

Preguntar sobre el código abierto en el editor

El agente tiene acceso al contenido del fichero activo (el que está en primer plano). Puedes preguntarle sin copiar nada:

"Explícame qué hace esta función"
"¿Hay algún posible error en este código?"
"Resume qué hace este fichero"

Si tienes varias pestañas abiertas y quieres preguntar sobre una concreta, menciona el nombre del fichero para evitar ambigüedades:

"Explícame qué hace el código del fichero fev_server.prg abierto en el editor"
"¿Qué función gestiona el login en chat_login.prg?"

Para preguntar sobre un fichero que no está abierto en el editor, indícalo también de forma explícita:

"Analiza el fichero fevscode.prg y dime cómo está estructurado"
"¿Qué hace la función AdjustLayout en fev_layout.prg?"

Pedir modificaciones de código

El agente puede modificar el fichero activo directamente. Algunos ejemplos habituales:

"Añade validación de que el campo usuario no esté vacío antes de hacer el login"
"Refactoriza la función XDServerStart separando la lógica de login en una función propia"
"Añade al final del fichero una nueva función GetUserCount que devuelva el número de usuarios en xdusers.json"
"Corrige el bug en remotePoll para que reintente si la respuesta está vacía"

8 Skills — fragmentos y plantillas de contexto

Las Skills son fragmentos de texto predefinidos (instrucciones, plantillas, fragmentos de código frecuentes) que puedes insertar en tus mensajes con un solo clic.

Para qué sirven

  • Instrucciones de sistema permanentes ("responde siempre en español", "actúa como experto en FiveWin…")
  • Plantillas de código que usas con frecuencia
  • Contexto del proyecto que quieres enviar al agente sin reescribirlo

Cómo usarlas

  1. Escribe /skill en el cuadro de mensaje o usa el botón de Skills del panel.
  2. Aparece la lista de skills disponibles.
  3. Selecciona una y su contenido se inserta en el mensaje.
Las Skills se guardan en el fichero xdskills.json junto al ejecutable. Puedes editarlo directamente con XDForCode para añadir o modificar skills.

9 Kanban — múltiples agentes en paralelo

El Kanban permite lanzar y coordinar múltiples agentes de IA simultáneamente, cada uno con su propia tarea.

Abrir el Kanban

Pulsa el icono de Kanban en la barra lateral. Se abre el tablero en el panel central.

Cómo funciona

  1. Crea una tarea en el tablero: asígnale nombre, descripción y agente.
  2. El agente arranca en su propia pestaña de terminal.
  3. El tablero muestra el estado: pendiente, en curso, completada.
  4. Puedes ver la conversación de cada agente en su pestaña individual.

Casos de uso

  • Lanzar un agente que refactoriza mientras otro escribe los tests.
  • Pedir a Claude, Gemini y Ollama la misma pregunta y comparar resultados.
  • Dejar tareas largas en segundo plano mientras trabajas en otra cosa.

10 Acceso remoto desde el navegador

XDForCode incluye un servidor web que permite acceder al chat con el agente desde cualquier navegador de tu red local.

Arrancar el servidor web

Pulsa el icono del servidor en la barra lateral. Cuando está activo la barra de estado inferior muestra Server :8003.

Acceder desde el navegador

http://localhost:8003

Verás la interfaz de chat idéntica al panel de XDForCode. Las respuestas llegan en tiempo real.

Acceso multiusuario con contraseña

Crea el fichero xdusers.json junto al ejecutable:

[
  { "user": "cristobal", "pass": "<SHA-256 de la contraseña>" }
]

Con este fichero presente, al acceder desde el navegador aparecerá una pantalla de inicio de sesión.

Sin xdusers.json, el acceso es libre (modo local, sin contraseña).

El explorador ocupa el panel izquierdo y muestra el árbol de directorios de la carpeta de trabajo actual.

AcciónResultado
Clic en un fichero de textoSe abre en el editor Monaco
Clic en un fichero binarioSe abre con la aplicación por defecto de Windows

El explorador se actualiza automáticamente cuando los agentes IA crean o modifican ficheros.

13 Preguntas frecuentes

¿Necesito instalar todos los agentes?

No. Con instalar uno ya puedes trabajar. Si no tienes claro cuál elegir, empieza con Ollama (gratuito, sin internet) o Claude Code.

¿El agente tiene acceso a mis ficheros?

El agente recibe como contexto el contenido del fichero que tienes abierto en el editor. No explora tus directorios automáticamente salvo que le des permiso explícito o uses herramientas MCP.

¿Puedo usar varios agentes a la vez?

Sí. Mediante el Kanban puedes tener varios agentes activos en paralelo, cada uno en su propia pestaña con su propia tarea.

¿El historial del chat queda guardado?

El historial de la sesión actual se mantiene mientras la aplicación está abierta. Al cerrar, el historial no se persiste por defecto.

¿Puedo usar XDForCode sin conexión a internet?

Sí, siempre que uses Ollama como agente. El resto requieren conexión para acceder a sus APIs en la nube.

¿Qué hago si el agente tarda demasiado?

Puedes cancelar la respuesta en cualquier momento con el botón Cancelar del panel de chat. El agente se detiene y puedes escribir de nuevo.

¿Cómo añado usuarios para el acceso remoto?

Crea el fichero xdusers.json junto al ejecutable con el formato indicado en la sección 10. Cada usuario necesita su nombre y el hash SHA-256 de su contraseña.
